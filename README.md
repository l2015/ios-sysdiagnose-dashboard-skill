# iPhone Sysdiagnose Analyzer

Analyze iPhone sysdiagnose diagnostic archives. Extract battery health, NAND flash SMART data, app usage metrics, crash logs, and more — generate a self-contained HTML report.

## What It Does

Parses an iPhone sysdiagnose `.tar.gz` file (generated by iOS: Settings → Privacy → Analytics → Start Sysdiagnose) and extracts:

- **Battery**: health percentage, charge cycles, capacity degradation, temperature trend
- **NAND Flash**: lifespan consumed, total read/write, PE cycles, bad blocks, power-on hours
- **Apps**: NAND write volume, screen time, energy consumption, memory peaks, GPS usage
- **Crashes**: Jetsam events, Safari crashes, disk write exceedance, CPU resource events
- **Network**: VPN/proxy extensions, network traffic per process
- **Device**: model, SoC, disk layout, partitions

Outputs a single self-contained HTML file with bento grid layout and SVG sparkline charts.

## Quick Start

```bash
pip install sysdiagnose

# 1. Parse the sysdiagnose archive
sysdiag create your_sysdiagnose.tar.gz
CASE_ID=$(sysdiag cases | tail -1 | awk '{print $1}')
sysdiag -c "$CASE_ID" parse all

# 2. Extract the archive for raw data access
mkdir -p /tmp/sd && tar xzf your_sysdiagnose.tar.gz -C /tmp/sd
BASE=$(find /tmp/sd -maxdepth 1 -type d -name "sysdiagnose_*" | head -1)

# 3. Extract structured data
python3 scripts/extract.py "$BASE" "cases/$CASE_ID/parsed_data" -o data.json

# 4. Generate report
python3 scripts/report.py data.json -o report.html
```

Open `report.html` in any browser.

## Requirements

- Python 3.10+
- [SAF (Sysdiagnose Analysis Framework)](https://github.com/EC-DIGIT-CSIRC/sysdiagnose) — `pip install sysdiagnose`
- `strings` command (pre-installed on macOS and Linux)

## CLI Reference

### extract.py

```
usage: extract.py [-h] [--max-points N] [-o OUTPUT] base_dir parsed_dir

positional arguments:
  base_dir              Extracted sysdiagnose directory
  parsed_dir            SAF parsed data directory

options:
  --max-points N        Max data points for time series (default: 200)
  -o, --output FILE     Output file (default: stdout)
```

### report.py

```
usage: report.py [-h] [-o OUTPUT] input

positional arguments:
  input                 JSON file from extract.py (or - for stdin)

options:
  -o, --output FILE     Output HTML file (default: stdout)
```

## Supported Devices

Tested with iPhone sysdiagnose archives from iOS 15-26. Works with any iPhone/iPad sysdiagnose that contains PowerLog data.

Model mapping (for display):
- iPhone14,2 → iPhone 13 Pro
- iPhone14,5 → iPhone 13
- iPhone15,2 → iPhone 14 Pro
- iPhone15,3 → iPhone 14 Pro Max
- iPhone16,1 → iPhone 15 Pro
- etc.

## Limitations

- NAND vendor/model not exposed in customer sysdiagnose (only Apple AST2)
- PowerLog tables may vary between iOS versions (some tables may be empty)
- Report text is in English only (no i18n yet)
